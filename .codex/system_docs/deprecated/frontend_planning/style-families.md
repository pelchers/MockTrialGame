# Style Families

## Primary Families (5)

These are the core style families with 6+ pass variants defined in `style-config.json`.

| 2 | stamp-grid-manifesto | Poster/broadsheet grid, stamps, numbered badges, editorial brutalism |
| 3 | terminal-command-line | CRT terminal, green phosphor, keyboard-first, CLI aesthetic |
| 4 | construction-zone-blueprint | Blueprint grid, safety orange, stencil typography, measurement annotations |
| 5 | protest-poster-collage | Torn paper, wheat-paste, magenta+yellow, DIY zine energy |
| 6 | warehouse-industrial-logistics | Corrugated steel, barcode labels, hazmat stripes, container grids |

## How Passes Differ Within a Family

Each pass within a family shares the same stylistic DNA but is structurally distinct across these dimensions:

1. **Layout Architecture**: Different uniqueness profile from the catalog (e.g., left rail vs. bottom dock vs. radial hub)
2. **Palette Overrides**: Different color expression within the family's tonal range
3. **Typography Overrides**: Different font selections within the family's typographic tradition
4. **Interaction Profile**: Different animations, transitions, and micro-interactions
5. **Content Persona**: Different visual metaphor and tonal flavor
6. **View Hints**: Different component patterns per view (e.g., kanban as columns vs. swimlanes vs. timeline)
7. **Anti-Repeat**: Hard constraints explicitly banning elements used in prior passes
